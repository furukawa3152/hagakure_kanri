# Amazon Q Developer CLI 勉強会 まとめ

## 1. 勉強会の目的
Amazon Q Developer CLI を使って簡単なゲーム開発とブログ記事作成を体験し、AWS公式のTシャツプレゼントキャンペーンに応募する。

## 2. 勉強会の流れ
1. **環境確認 (21:50頃)**
   - 画面共有、音声の確認
   - チャットはSlackを使用

2. **Amazon Q Developerの概要説明 (21:50頃)**
   - 自然言語でAWSの操作やプログラミングをサポートするAIアシスタント
   - ターミナル上で使用可能

3. **ハンズオン①：ゲーム開発 (22:00 - 22:20)**
   - `q chat` でチャットを開始
   - 日本語で指示を出し、`pygame` を利用したタイピングゲームを作成
     - ディレクトリ作成 (`mkdir game`)
     - ディレクトリ移動 (`cd game`)
     - ゲーム作成の指示出し
       - 「`pygame`を使って、電車好きが遊べるタイピングゲームを作ってください」
   - **発生した問題と解決策**
     - **問題1：** ゲーム内の日本語が文字化け (`?`で表示される)
     - **解決策1：** 「日本語に対応したフォントを使ってください」と指示し、フォントを修正
     - **問題2：** ゲームが自動で始まらない
     - **解決策2：** 一度ウィンドウを閉じて再度実行することで対応
     - **問題3：** 効果音の追加
     - **解決策3：** 「効果音を追加してください」と指示。AIが`pygame`のサウンドエンジンを使い、サイン波で音を生成しようと試みるが、意図した通りにはならなかった。

4. **ハンズオン②：ブログ記事作成 (22:20 - 22:50)**
   - ゲーム開発の体験をブログにまとめる
   - HAGAKUREプログラミング塾のブログシステム `Scroll` を利用
   - **注意点**
     - ブログのURL(スラッグ)は、日本語ではなく半角英数字で入力する必要がある。

5. **Q&A (22:50 - )**
   - 参加者からの質疑応答

## 3. 参加者の様子 (チャットログより)
- `trustall` コマンドの確認と実行
- `pygame` でのゲーム作成がブラウザベースではなく、専用のウィンドウが立ち上がることに驚きの声
- AIが自動で`pygame`のインストールや環境設定を行う様子に感心の声
- AIが生成した効果音が期待と異なっていた
- ターミナルが予期せず落ちる参加者がいたが、スペックの問題の可能性
